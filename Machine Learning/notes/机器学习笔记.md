遗忘门f：控制长期状态遗忘一些不重要的信息。根据当前输入x和上一时刻输入状态h，对长期状态c进行缩减

输出门o：控制长期状态中信息的输出程度。根据当前输入x和上一时刻输入状态h，判断长期状态c中哪些需要输出到h

## 自注意力模型

![1767882782432](images/ML/1767882782432.png)![1767882799830](images/ML/1767882799830.png)![1767882811060](images/ML/1767882811060.png)

在这个过程中，相当于通过学习得到QKV的矩阵参数后，对输入的token进行计算得到QKV，并通过公式计算，得到一个变换矩阵，通过这个矩阵，可以对当前的token输入变换得到其得到带上上下文的新表示

## 多头注意力模型

![1767883529369](images/ML/1767883529369.png)

并行地使用多组独立的、可学习的 Q、K、V 投影矩阵，对输入进行多次不同的注意力计算，得到多个上下文表示（“多个视角”），然后再经过一个可学习的线性变换，将得到的这些新表示进行融合将它们拼接起来，形成更丰富、更全面的最终表示。

```
Attention scores before mask:

  [w21, w22, w23],
  [w31, w32, w33] ]

After masking (set future to -∞):
[ [w11,  -∞,  -∞],

  [w31, w32, w33] ]
```

![1767884500437](images/ML/1767884500437.png)![1767884521874](images/ML/1767884521874.png)

注意，在推理时，仍需要进行掩码策略，区别是需要进行多次解码器操作，即将不断扩充的结果重新进行解码预测下一个token并继续对结果进行扩展。

如果推理完全正确的话，那么训练时的解码器输入与最后一次推理时的解码器输入完全相同，但是其输出利用方式并不一样，由于每一行理应是预测下一个token用的，于是在训练时，我们对每一行都可以计算损失，同时这也是并行进行的，而在训练时，我们只是为了预测下一个token，所以只需要利用最后一行的结果。
