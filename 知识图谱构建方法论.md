## 1. 核心概念

### 1.1 知识图谱三原则

| 原则       | 说明                               |
| -------- | -------------------------------- |
| **原子化**  | 每个笔记只讲一个概念，粒度均匀                  |
| **树状优先** | 主体结构为辐射树（根→章→节→知识点），在此之上可添加语义跨链接 |
| **一致命名** | 文件、标签、链接格式统一，确保图谱节点可读            |

### 1.2 三种节点类型

```
根节点 (Root)            — 知识领域的入口，链接到所有一级章节
  │
章节导航节点 (Hub)       — 一个知识单元的目录，链接到下属知识点 + 回链根节点
  │
原子笔记 (Leaf)          — 具体知识点，不包含任何出链 wiki-link
```

### 1.3 链接在图谱中的可见性

| 链接类型 | 语法 | 全局图谱 | 局部图谱 | 用途 |
|---------|------|---------|---------|------|
| 父子结构链接 | `[[path/file]]` | ✅ 可见 | ✅ 可见 | Hub→原子笔记、根→Hub |
| 上一个/下一个 | `[text](file.md)` | ❌ 不可见 | ❌ 不可见 | 章节内顺序导航 |
| 相关知识点 | `[text](path/file.md)` | ❌ 不可见 | ❌ 不可见 | 跨学科/跨章节语义关联 |

## 2. 项目文件结构标准

### 2.1 标准目录布局

> ⚠️ 以下结构是经过 CSAPP + 6 个学科（Algorithms、Data Structures、Discrete Math、Info Theory、Machine Learning、Signals and System）大规模实践后验证的最佳实践。

```
<Vault root>/
├── .obsidian/
│   └── graph.json                      ← 图谱颜色组等配置
├── _知识图谱构建方法论.md               ← 本文档（可选，放在 vault 根）
│
├── <项目根目录>/                        ← 如 Algorithms、CSAPP（学科英文名）
│   ├── <项目根目录>.md                  ← 根节点文件，如 CSAPP.md
│   ├── 01-<章节中文名>/                 ← 2位数字前缀 + 中文名
│   │   ├── <章节中文名>.md              ← 章节导航节点（Hub），如 算法分析.md
│   │   ├── <X.Y> <知识点名>.md          ← 原子笔记，必须带 X.Y 前缀，如1.1
│   │   ├── <X.Y> <知识点名>.md
│   │   └── ...
│   ├── 02-<章节中文名>/
│   │   └── ...
│   ├── ...
│   ├── images/                         ← 所有图片扁平存放，不分章节子目录
│   │   ├── 001_章节_概念名.png          ← 3位序号_章节_描述.png
│   │   ├── 002_章节_概念名.png
│   │   └── ...
│   ├── code/                           ← 代码文件归档（可选，如 .cpp/.exe）
│   ├── notes/                          ← 原始笔记备份（可选，仅参考不修改）
│   └── slides/                         ← 原始 PDF 课件（可选）
```

### 2.2 目录与文件命名规则

| 要素 | 规则 | 示例 |
|------|------|------|
| 章节目录 | `<2位数字>-<中文名>` | `01-算法分析`、`03-傅里叶变换` |
| 章节导航文件（Hub） | `<中文章节名>.md`（与目录名去掉数字前缀一致） | `算法分析.md`、`傅里叶变换.md` |
| 原子笔记 | **必带** `<X.Y> <名称>.md`，X=章号 Y=序号 | `1.1 复杂度分析.md`、`2.3 整型数表示.md` |
| 根节点 |  `<项目名>.md`（与目录名一致） | `Algorithms.md`、`CSAPP.md` |
| 图片 |  `images/` 目录扁平存放 | `images/` |
| 代码归档 | `code/` 目录 | `code/` |

### 2.3 原子笔记命名规范

> ⚠️ 这是最重要的规范之一。所有原子笔记**必须**包含 `X.Y` 数字前缀。

**格式**：`<章号>.<序号> <知识点描述>.md`

| 章节 | 正确示例 | 错误示例 |
|------|---------|---------|
| 01-算法分析 | `1.1 复杂度分析.md` | `复杂度分析.md` ✗ |
| 02-信息的表示和处理 | `2.3 整型数表示.md` | `整型数.md` ✗ |
| 03-图论 | `3.5 哈密顿图.md` | `3.5 哈密顿图.md` ✓ |

**排序规则**：按 (章号, 序号) 自然排序，即 `2.9 → 2.10`（非字符串排序的 `2.1 → 2.10`）

**序号范围**：每个章节内从 1 开始连续编号，中间不可跳跃，不可重复。

**知识点顺序**：必须按照原始笔记的**循序渐进原则**排列（从基础到进阶），而非随意排序。

### 2.4 Hub 文件命名

章节导航文件（Hub）的名称**必须**与章节目录名匹配：

| 目录名 | Hub 文件名 |
|--------|-----------|
| `01-算法分析` | `算法分析.md` |
| `03-傅里叶变换` | `03-傅里叶变换.md`（也可省略数字前缀） |

> **不要使用 `_moc.md` 作为 Hub 文件名**，否则图谱中所有 Hub 节点都显示为 `_moc`，无法区分。

### 2.5 根节点命名

根节点文件名为 `<项目名>.md`，放在项目根目录。命名规则：

| 项目 | 目录名 | 根节点文件名 |
|------|--------|-------------|
| CSAPP | `CSAPP/` | `CSAPP.md` |
| 算法 | `Algorithms/` | `Algorithms.md` |
| 信息论 | `Info Theory/` | `Info Theory.md` |

## 3. 文件内部格式规范

### 3.1 根节点文件

**位置**：`<项目根目录>/<项目名>.md`

**格式**：

```markdown
---
tags: [<project-tag>, map-of-content, root]
---

# <项目名> 知识图谱

> <一句话描述>

## 章节导航

| # | 章节 | 核心主题 |
|---|------|---------|
| 第1章 | [[01-算法分析/算法分析\|算法分析]] | 复杂度、递归、摊还分析 |
| 第2章 | [[02-动态规划/动态规划\|动态规划]] | 矩阵链乘、LCS、OBST、背包 |
| ... | ... | ... |
```

**规范要点**：
- `tags` 必须包含 `<project-tag>`、`map-of-content`、`root` 三个标签
- 使用 Wiki-link 链接到每个章节 Hub（`[[目录名/文件名]]`）
- 可以有表格或列表形式

### 3.2 章节导航节点（Hub）

**位置**：`<章节目录>/<章节中文名>.md`

**格式**：

```markdown
---
tags: [<project-tag>, moc, <ch-tag>]
aliases: [第<N>章, <章节中文名>]
---

# 第<N>章 <章节中文名>

## 本章知识点

- [[<项目名>|<章节中文名>]]                       ← 回链到根节点
- [[<章节目录>/<笔记1>|<显示名>]]
- [[<章节目录>/<笔记2>|<显示名>]]
- ...
```

**规范要点**：
- `tags` 必须包含 `<project-tag>`、`moc`、`<ch-tag>`（如 `algo-ch1`）
- 第一条链接必须是**回链到根节点**
- 后续排列所有原子笔记的链接
- `aliases` 可选，便于在图谱中显示友好名称

**各学科章节标签对照表**：

| 学科 | project-tag | ch-tag 格式 | 示例 |
|------|-------------|-------------|------|
| CSAPP | `csapp` | `chapter{N}` | `chapter1` |
| Algorithms | `algorithms` | `algo-ch{N}` | `algo-ch1` |
| Data Structures | `data-structures` | `ds-ch{N}` | `ds-ch1` |
| Discrete Math | `discrete-math` | `dm-ch{N}` | `dm-ch1` |
| Info Theory | `info-theory` | `it-ch{N}` | `it-ch1` |
| Machine Learning | `machine-learning` | `ml-ch{N}` | `ml-ch1` |
| Signals and System | `signals-and-system` | `ss-ch{N}` | `ss-ch1` |

### 3.3 原子笔记

**位置**：`<章节目录>/<X.Y> <知识点名>.md`

**完整格式示例**：

```markdown
---
created: 2026-07-20
tags: [algo-ch1, algorithms, 复杂度, 时间复杂度]
---

# 复杂度分析

复杂度分析是算法分析的核心，用于评估算法在输入规模增大时的效率变化趋势。

## 常见复杂度类型

- **常数时间 O(1)**：与输入规模无关
- **对数时间 O(log n)**：二分法查找
- **线性时间 O(n)**：单层遍历
- **平方时间 O(n²)**：双层嵌套循环

## 链接到

- 上一个知识点：无（本章第一个知识点）
- 下一个知识点：[1.2 递归函数](1.2%20%E9%80%92%E5%BD%92%E5%87%BD%E6%95%B0.md)
- 相关知识点：[7.11 Dijkstra算法](07-%E5%9B%BE/7.11%20Dijkstra%E7%AE%97%E6%B3%95.md)
```

**规范要点**：

| 部分 | 要求 |
|------|------|
| 文件名 | 必须包含 `<X.Y> ` 前缀，如 `1.1 复杂度分析.md` |
| `tags` | 必须包含 `<ch-tag>` 和 `<project-tag>`，可选关键词标签 |
| 正文标题 | `# <知识点名>`（不含 X.Y 前缀） |
| 正文 | 中文技术内容，包含定义、解释、关键公式 |
| 图片引用 | 使用 `![[path/to/image.png]]` 格式 |
| **上一个/下一个** | `[X.Y 名称](file.md)` — 显示文本**包含** X.Y 前缀 |
| **相关知识点** | `[X.Y 名称](path/file.md)` — 显示文本同样包含 X.Y 前缀 |
| 链接语法 | 使用**标准 Markdown 链接** `[text](file.md)` — 图谱不可见 |
| 禁止 | **不得包含任何 wiki-link**（`[[...]]`），否则图谱中会产生多余的边 |

> **关键规则**：导航链接的显示文本必须包含 `X.Y` 前缀，如 `[1.2 递归函数](1.2%20%E9%80%92%E5%BD%92%E5%87%BD%E6%95%B0.md)`，而不仅仅是 `[递归函数]`。

**导航链接的图谱行为**：

| 链接类型 | 语法 | 全局图谱 | 局部图谱 |
|---------|------|---------|---------|
| Hub→原子笔记 | `[[path/file]]` wiki-link | ✅ 显示 | ✅ 显示 |
| 上一个/下一个 | `[text](file.md)` markdown 链接 | ❌ 不显示 | ❌ 不显示 |
| 相关知识点 | `[text](path/file.md)` markdown 链接 | ❌ 不显示 | ❌ 不显示 |

**知识点顺序原则**：原子笔记必须按照原始笔记的**循序渐进**原则排列。

```python
# 知识点编号规则
# X = 章号（来自目录名的前两位数字，如 01 → 1, 07 → 7）
# Y = 知识点序号（从 1 开始连续编号）

# 排序方式：自然排序（numeric sort），非字典序
# 正确：1.1, 1.2, ..., 1.9, 1.10, 1.11  (numeric)
# 错误：1.1, 1.10, 1.11, 1.2, 1.3       (lexicographic)
```

## 4. 图片处理规范

### 4.1 图片存储位置

所有图片**扁平存放**在项目根目录的 `images/` 文件夹中，**不分章节子目录**。

```
<项目>/
├── images/                         ← OCR 只处理这个文件夹
│   ├── 001_复杂度分析_常见复杂度类型.png
│   ├── 002_复杂度分析_常见复杂度类型.png
│   └── ...
├── notes/images/                   ← ⚠️ 原始笔记备份，OCR 绝不能碰
│   └── 1762596851160.png           ← （保留原始时间戳文件名不变）
```

> ⚠️ **关键规则**：OCR 管道**只能**处理 `<项目>/images/` 目录下的图片。
> `notes/` 目录是原始笔记的只读备份，其中的 `notes/images/` 保留原始图片**不动**。
> 如果不加限制地遍历整个项目目录，会误处理 `notes/images/` 并在其中生成重命名后的重复图片。

### 4.2 图片文件命名规则

**格式**：`<3位序号>_<所属章节或知识点关键词>_<描述>.png`

| 原始名称 | 处理后名称 |
|---------|-----------|
| `1762596851160.png` | `001_复杂度分析_常见复杂度类型.png` |
| `1767334275814.png` | `001_一阶逻辑基本概念_公式分类.png` |

**命名规范**：
- 文件名由上下文信息自动生成（笔记文件名 + 笔记中小标题）
- 避免路径中带空格（尤其是学科名如 `Data Structures`）导致的处理问题
- 文件名保持 UTF-8 编码

### 4.3 图片引用格式

图片在笔记中使用 **Obsidian embed 语法**：

```markdown
![[path/to/image.png]]
```

> **正确格式**：`![[Algorithms/images/001_复杂度分析_常见复杂度类型.png]]`
>
> ❌ **错误格式对照**：
> - `!path/to/image.png`（缺少 `[[ ]]`）
> - `![[[[path/to/image.png]]]]`（括号重复）
> - `![path/to/image.png]`（单括号 Markdown 语法，Obsidian 不识别）
>
> ⚠️ 学科名中的**空格**（如 `Data Structures`、`Discrete Math`）是正则匹配的常见陷阱，务必测试确认。

```python
def generate_image_name(index, note_name, context_heading):
    """生成描述性图片文件名"""
    safe_note = sanitize(note_name)       # 清理特殊字符
    safe_context = sanitize(context_heading)
    return f"{index:03d}_{safe_note}_{safe_context}.png"
```

### 4.2 OCR 处理流程

#### ⚠️ 重要：路径限制规则

OCR 管道**只能**扫描 `images/` 目录，绝不能遍历整个项目目录（否则会误处理 `notes/images/` 备份）。

```python
# ✅ 正确：只扫描 images/
IMAGE_DIR = os.path.join(BASE_DIR, "images")
image_files = [f for f in os.listdir(IMAGE_DIR) if f.endswith(('.png', '.jpg'))]

# ❌ 错误：遍历整个项目目录（会误伤 notes/images/）
# for root, dirs, files in os.walk(BASE_DIR):  ← 禁止这样！
```

#### 完整 OCR 处理管道

```python
import subprocess, tempfile, os, re

BASE_DIR = "<项目根目录>"   # 如 D:/DOC/Algorithms
IMAGE_DIR = os.path.join(BASE_DIR, "images")

# ⚠️ 只扫描 images/ 目录，不碰 notes/ 等其他目录
image_files = sorted(os.listdir(IMAGE_DIR))
image_files = [f for f in image_files if f.endswith(('.png', '.jpg'))]

def safe_ocr(image_path, lang="chi_sim+eng"):
    """使用临时文件避免 Windows 编码问题"""
    tmp = tempfile.NamedTemporaryFile(suffix='.txt', delete=False, mode='w', encoding='utf-8')
    tmp_path = tmp.name
    tmp.close()

    base = tmp_path.replace('.txt', '')
    subprocess.run(
        ["tesseract", image_path, base, "-l", lang, "--psm", "6"],
        capture_output=True, timeout=30
    )

    result_path = base + '.txt'
    text = ""
    if os.path.exists(result_path):
        with open(result_path, 'r', encoding='utf-8', errors='replace') as f:
            text = f.read().strip()

    if len(text) < 10:
        subprocess.run(
            ["tesseract", image_path, base, "-l", "eng", "--psm", "6"],
            capture_output=True, timeout=30
        )
        if os.path.exists(result_path):
            with open(result_path, 'r', encoding='utf-8', errors='replace') as f:
                text = f.read().strip()

    try:
        os.unlink(tmp_path)
        if os.path.exists(result_path):
            os.unlink(result_path)
    except:
        pass

    return text

# 对每张图片执行 OCR
for img_file in image_files:
    img_path = os.path.join(IMAGE_DIR, img_file)
    text = safe_ocr(img_path)
    # ... 根据 OCR 结果分类处理 ...
```

#### OCR 安全提取函数

同上节定义的 `safe_ocr` 函数。

#### 图片分类逻辑

| 分类 | 判断条件 | 处理方式 |
|------|---------|---------|
| **文字可替换** | OCR 结果 ≥ 10 字符有意义文字 | 将文字提取到笔记中，删除图片文件 |
| **纯图示** | 无文字或极少文字 | 保留图片，描述性重命名 |
| **复杂图文** | 有文字但含图表框线 | 保留图片并重命名 |


### 5.3 标签去重脚本核心逻辑

```python
def normalize_tags(content):
    """统一标签格式：去重、排序、确保必要标签存在"""
    import re

    fm_match = re.match(r'^---\s*\n(.*?)\n---', content, re.DOTALL)
    if not fm_match:
        return content

    yaml = fm_match.group(1)
    tags_match = re.search(r'tags:\s*\[([^\]]*)\]', yaml)
    if not tags_match:
        return content

    # 解析、去重（保留顺序）
    tags = tags_match.group(1).split(',')
    seen = set()
    unique_tags = []
    for tag in tags:
        t = tag.strip()
        if t and t not in seen:
            seen.add(t)
            unique_tags.append(t)

    # 写回
    yaml_new = re.sub(
        r'tags:\s*\[([^\]]*)\]',
        f"tags: [{', '.join(unique_tags)}]",
        yaml
    )
    return content.replace(yaml, yaml_new)
```

## 6. 链接架构规范

### 6.1 辐射树结构

整个图谱的核心结构是**分层辐射树**，保证图谱清晰可读：

```
                    ┌─ 根节点 ─┐
                   /     |      \
                  ▼      ▼       ▼
            ┌─ Hub1 ─┐ ┌─ Hub2 ┐ ┌─ Hub3 ─┐ ...
           /    |    \  / |  \   /  |  \
          ▼     ▼     ▼ ▼  ▼  ▼  ▼  ▼  ▼
      笔记1.1 笔记1.2 笔记1.3 ...（共46章×平均7篇=306篇原子笔记）
```

**链接规则**：

| 连接 | 方向 | 语法 | 图表现 |
|------|------|------|--------|
| 根→Hub | 出链 | `[[01-目录名/Hub名]]` | ✅ |
| Hub→根 | 回链 | `[[项目名]]` | ✅ |
| Hub→原子笔记 | 出链 | `[[目录/笔记\|显示名]]` | ✅ |

### 6.2 导航链接

在每个原子笔记底部添加 **上一个/下一个/相关知识点** 导航，使用标准 Markdown 链接语法。

```markdown
## 链接到

- 上一个知识点：无（本章第一个知识点）
- 下一个知识点：[1.2 递归函数](1.2%20%E9%80%92%E5%BD%92%E5%87%BD%E6%95%B0.md)
- 相关知识点：[7.11 Dijkstra算法](07-%E5%9B%BE/7.11%20Dijkstra%E7%AE%97%E6%B3%95.md)
```

**关键规则**：

| 规则 | 说明 |
|------|------|
| 语法 | 使用 `[text](encoded_path.md)` 格式，**禁用** `[[path]]` |
| 显示文本 | **必须包含 X.Y 前缀**，如 `[1.2 递归函数]` 而非 `[递归函数]` |
| 路径编码 | URL 编码，中文 → 百分比编码，空格 → `%20` |
| 上一个/下一个 | 按笔记文件名**自然排序**（numeric sort）确定顺序 |
| 相关知识点 | 仅在不同章节/学科间建立；同一章节内不连接 |
| 跨学科链接 | 路径包含完整相对路径，如 `07-图/7.11 Dijkstra算法.md` |
| 第一个笔记 | 显示 `无（本章第一个知识点）` |
| 最后一个笔记 | 显示 `无（本章最后一个知识点）` |

**排序实现**：

```python
def numeric_sort_key(filename):
    """按 (章号, 序号) 自然排序"""
    import re
    m = re.match(r'(\d+)\.(\d+)', filename)
    if m:
        return (int(m.group(1)), int(m.group(2)))
    return (9999, 9999)
```

### 6.3 跨边界语义链接

**原则**：只在**不同章节**或**不同学科**之间建立语义关联。同一章节内的笔记不互相关联（避免图谱密集连接）。

**算法**：

```python
def build_cross_boundary_links(note_index):

    for note_key, note_data in note_index.items():

            other_terms = other_data['terms']
            shared = terms & other_terms

            if len(shared) >= 3:  # 至少共享3个特有术语
                scored.append((len(shared), other_key))

        # 取最相关的1-2个
        scored.sort(key=lambda x: -x[0])
        top = scored[:2]

```

**术语提取来源**：
- **加粗文本** `**term**`
- **小标题** `## Heading`
- **英文技术名词**（大写开头的词、缩写词）
- **标签中的关键词**

```python
def validate_subject(vault_root, subject):
    """验证单个学科的图谱结构"""
    base = os.path.join(vault_root, subject)
    issues = []

    # 1. 根节点检查
    root_file = os.path.join(base, f"{subject}.md")
    if not os.path.exists(root_file):
        issues.append("根节点文件缺失")

    # 2. 章节 Hub 检查
    chapters = [d for d in os.listdir(base) if re.match(r'^\d', d)]
    for ch in chapters:
        ch_dir = os.path.join(base, ch)
        hub_name = f"{ch[3:]}.md" if '-' in ch else f"{ch}.md"
        hub_path = os.path.join(ch_dir, hub_name)
        if not os.path.exists(hub_path):
            issues.append(f"{ch}: Hub 文件 {hub_name} 缺失")

    # 3. 原子笔记检查
    for ch in chapters:
        ch_dir = os.path.join(base, ch)
        for f in os.listdir(ch_dir):
            if not f.endswith('.md') or f == hub_name:
                continue
            fp = os.path.join(ch_dir, f)
            with open(fp, 'r', encoding='utf-8') as mdf:
                content = mdf.read()
            # 非图片 wiki-link 检查
            links = re.findall(r'(?<!!)\[\[([^\]]*)\]\]', content)
            if links:
                issues.append(f"{ch}/{f}: 含有 {len(links)} 个出链")
            # 图片引用格式检查
            for m in re.finditer(r'!([^\s!]+\.png)', content):
                if not m.group(0).startswith('!['):
                    issues.append(f"{ch}/{f}: 图片格式错误 {m.group(0)[:30]}")

    # 4. 导航链接格式检查
    for ch in chapters:
        ch_dir = os.path.join(base, ch)
        for f in os.listdir(ch_dir):
            if not f.endswith('.md'): continue
            fp = os.path.join(ch_dir, f)
            with open(fp, 'r', encoding='utf-8') as mdf:
                content = mdf.read()
            if '## 链接到' in content:
                sec = content.split('## 链接到')[1]
                if '[[' in sec:
                    issues.append(f"{ch}/{f}: 导航区有 wiki-link")

    return issues
```

**检查清单**：
- [ ] 根节点存在且标签正确
- [ ] 每个章节有对应的 Hub 文件
- [ ] Hub 标签包含 `moc`
- [ ] 原子笔记标签包含 `<ch-tag>` 和 `<project-tag>`
- [ ] 原子笔记无 wiki-link 出链
- [ ] 所有图片引用格式为 `![[path.png]]`
- [ ] 导航链接使用 Markdown 格式 `[text](file.md)`
- [ ] 导航区无 wiki-link
- [ ] 无 `.ocr.txt` 等临时文件残留

**现象**：Python `print()` 时 `UnicodeEncodeError: 'gbk' codec can't encode character`

**解决方案**：
- 脚本中避免使用 emoji、✓、✗ 等 Unicode 符号，改用 ASCII（`OK`, `DONE`）
- 或设置 `sys.stdout.reconfigure(encoding='utf-8')`
- 或重定向输出到文件：`python script.py > output.log`

### 9.2 Tesseract 编码崩溃

**现象**：`subprocess.run(capture_output=True, text=True)` 执行 Tesseract 时崩溃

**原因**：Windows 上的 Python 用 GBK 解码 Tesseract 的 UTF-8 输出

**解决方案**：写入临时文件再读取（见 [OCR 安全提取函数](#ocr-安全提取函数)）

### 9.3 图片格式错误（最易犯）

**正确**：`![[path/to/image.png]]`
**错误**：`!path/to/image.png`（缺少括号）
**错误**：`![[[[path/to/image.png]]]]`（括号重复）
**错误**：`![path/to/image.png]`（单括号）

⚠️ 文件路径中有空格（如 `Data Structures/images/`）时，正则匹配特别容易出错，务必仔细测试。

### 9.4 导航链接不能使用 wiki-link

除非你想在图谱中看到它们。所有导航（上一个/下一个/相关知识点）必须使用 `[text](file.md)` 格式。

### 9.5 Hub 文件识别错误

生成 Hub 内容时，必须正确识别哪个文件是 Hub（匹配章节目录名）。错误的识别会把原子笔记内容覆盖为 Hub 内容。

### 9.6 OCR 误处理 notes/images/ 备份

**现象**：`notes/images/` 中生成了与原始图片重名的文件，或生成了 `001_xxx.png` 这样的重命名文件，造成 `notes/` 膨胀且包含重复图片。

**原因**：OCR 管道遍历了整个项目目录（含 `notes/`），把 `notes/images/` 中的图片也进行了 OCR 和重命名。

**预防**：
- OCR 处理时**只扫描** `images/` 目录，不要 `os.walk` 整个项目目录
- `notes/` 目录视为**只读备份**，任何处理脚本都不应修改其内容

**修复**（如果已经发生）：
```bash
# 删除 notes/images/ 中被重命名的文件（保留原始时间戳文件即可）
# 具体哪些是原始的、哪些是 OCR 生成的，需要根据文件名判断
# 原始：1762596851160.png（纯数字时间戳）
# OCR 生成：001_xxx.png（描述性名称）

# 安全做法：只删除描述性命名的文件
find <项目>/notes/images -name "*_*.png" -delete
```

### 9.7 数学公式未用 LaTeX 包裹

**现象**：笔记中的公式显示为普通文本（如 `H(X) = -∑p(x)log p(x)`），无法渲染为数学公式。

**原因**：直接使用了 Unicode 数学符号（∑、∫、α、β 等），而没有用 `$...$` 包裹。

**规则**：所有数学表达式必须用 LaTeX 包裹：
- 行内公式：`$公式$`
- 块公式：`$$公式$$`

**自动修复脚本**：见附录脚本模板。

### 9.8 重复运行导致标签或链接叠加

脚本应具有**幂等性**：运行多次结果一致。
- 写标签前去重
- 写链接前删除旧的
- 操作前检查是否已存在

### 9.9 图片处理与原子化的顺序错误

**现象**：原子笔记先写好了，图片后才进行 OCR 和重命名，导致笔记中引用的图片路径反复变更，或图片内容未被充分理解导致知识点拆分不准确。

**原因**：流程顺序搞反了——在原子化之前没有先处理原笔记中的图片。

**正确顺序**：
```
步骤1: 读取原始笔记
步骤2: OCR 原始笔记中的所有图片  ← 这一步必须在这里
步骤3: 根据 OCR 结果决定替换/保留图片
步骤4: 现在原笔记的内容已经完全可读，再拆分知识点
步骤5: 原子笔记中引用的 images/ 图片已经是处理好的最终版本
```

**效果**：
- ✅ 图片中的关键信息不会丢失（已被 OCR 文本化）
- ✅ 原子笔记中的图片引用路径一次确定，不需要反复修改
- ✅ notes/ 始终保持只读备份状态

### 11.1 批量图片重命名 + 引用更新

```python
import os, re

VAULT = "D:/DOC"
SUBJECT = "MySubject"

# 1. 构建 旧名→新名 映射
rename_map = {}

for root, dirs, files in os.walk(os.path.join(VAULT, SUBJECT)):
    for f in files:
        if not f.endswith('.md'): continue
        filepath = os.path.join(root, f)
        with open(filepath, 'r', encoding='utf-8') as mdf:
            content = mdf.read()

        # 找到所有图片引用，提取上下文
        for m in re.finditer(r'!\[\[[^\]]+/([^]]+\.png)\]\]', content):
            old_name = m.group(1)
            # 获取标题上下文
            pre_text = content[:m.start()]
            headings = re.findall(r'^#{1,3}\s+(.+)$', pre_text, re.MULTILINE)
            context = headings[-1] if headings else "general"

            # 生成新名称
            note_name = f.replace('.md', '')
            safe_context = re.sub(r'[<>:"/\\|?*#]', '', context)[:30]
            new_name = f"{len(rename_map)+1:03d}_{note_name}_{safe_context}.png"
            rename_map[old_name] = new_name

# 2. 重命名文件并更新引用
for old_name, new_name in rename_map.items():
    old_path = os.path.join(VAULT, SUBJECT, "images", old_name)
    new_path = os.path.join(VAULT, SUBJECT, "images", new_name)
    if os.path.exists(old_path):
        os.rename(old_path, new_path)
```

### 11.2 原子笔记链接剥离

```python
def strip_atomic_links(content):
    """移除原子笔记中的所有 wiki-link，保留图片嵌入"""
    # 保留 frontmatter
    parts = content.split('---', 2)
    if len(parts) >= 3:
        fm = parts[1]
        body = parts[2]
        # 移除 [[...]]，但保留 ![[...]]
        body = re.sub(r'(?<!!)\[\[([^\]]*)\]\]', '', body)
        body = re.sub(r'\n{3,}', '\n\n', body)
        return f"---{fm}---{body.strip()}\n"
    return content
```

### 11.3 Hub 内容生成

```python
def build_hub(subject, chapter_dir, root_tag):
    """生成章节 Hub 文件的内容"""
    ch_name = os.path.basename(chapter_dir)
    ch_topic = ch_name[3:] if '-' in ch_name else ch_name
    ch_num = ch_name[:2].lstrip('0')
    short = get_subject_short(subject)
    ch_tag = f"{short}-ch{ch_num}"

    files = sorted(os.listdir(chapter_dir))
    atomic_notes = [f for f in files if f.endswith('.md') and f != f"{ch_topic}.md"]

    content = "---\n"
    content += f"tags: [{subject.lower()}, moc, {ch_tag}]\n"
    content += f"aliases: [第{ch_num}章, {ch_topic}]\n"
    content += "---\n"
    content += f"# 第{ch_num}章 {ch_topic}\n\n"
    content += "## 本章知识点\n\n"
    content += f"- [[{subject}|{ch_topic}]]\n"  # 回链根

    for note in atomic_notes:
        note_name = note.replace('.md', '')
        content += f"- [[{ch_name}/{note_name}|{note_name}]]\n"

    return content

def get_subject_short(subject):
    """学科 → 短标签"""
    mapping = {
        "Algorithms": "algo", "Data Structures": "ds",
        "Discrete Math": "dm", "Info Theory": "it",
        "Machine Learning": "ml", "Signals and System": "ss",
        "CSAPP": "chapter",
    }
    return mapping.get(subject, subject.lower()[:3])
```

### 11.4 跨边界语义链接

```python
def extract_key_terms(body):
    """提取技术关键词"""
    terms = set()

    # 英文技术名词（大写开头的词）
    eng_tech = re.findall(r'\b[A-Z][a-z]{2,}\b', body)
    for t in eng_tech: terms.add(t.lower())

    # 全大写缩写
    for a in re.findall(r'\b[A-Z]{2,}\b', body):
        terms.add(a.lower())

    # 加粗术语
    for b in re.findall(r'\*\*([^*]+)\*\*', body):
        terms.add(b.strip().lower())

    # 小标题
    for h in re.findall(r'^#{1,3}\s+(.+)$', body, re.MULTILINE):
        terms.add(h.strip().lower())

    return terms

def find_related(note_index, subject, chapter, note_file):
    """找跨边界关联"""
    terms = note_index[(subject, chapter, note_file)]['terms']
    scored = []

    for (s, ch, nf), nd in note_index.items():
        if s == subject and ch == chapter: continue
        if nf == note_file: continue

        shared = terms & nd['terms']
        if len(shared) >= 3:
            scored.append((len(shared), ch, nf))

    scored.sort(key=lambda x: -x[0])
    return scored[:2]
```
