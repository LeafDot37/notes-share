# 🗺️ 贡献者指南 · 笔记规范

> 本文档面向希望向本项目贡献笔记的贡献者。
> 请遵循以下规范，确保知识库的一致性和可维护性。

---

## 目录结构规范

```
Notes/
├── index.md                     ← 知识库首页
├── 知识图谱构建方法论.md         ← 本文档
│
├── <学科英文名>/                 ← 如 Algorithms、CSAPP
│   ├── <学科英文名>.md           ← 学科根节点（入口）
│   ├── index.md                 ← 与根节点内容一致，用于 Explorer 点击目录时直达
│   ├── 01-<章节名>/             ← 数字前缀 + 中文名
│   │   ├── index.md             ← 章节导航页（与 Hub 内容一致）
│   │   ├── <章节中文名>.md     ← 章节 Hub，仅本地使用（不上传网站）
│   │   ├── <X.Y> <知识点>.md    ← 原子笔记，带编号
│   │   └── ...
│   ├── 02-<章节名>/
│   │   └── ...
│   ├── images/                  ← 图片资源（可选）
│   └── notes/                   ← 原始长篇笔记（不上传至网站）
│
├── Others/                      ← 其他辅助文档
└── README.md
```

## 文件命名规则

| 要素 | 规则 | 示例 |
|------|------|------|
| 学科根目录 | 英文名 | `Algorithms/`、`CSAPP/` |
| 章节目录 | `<2位数字>-<中文名>` | `01-算法分析` |
| 学科根节点 | `<学科名>.md`（同时创建 `index.md`） | `Algorithms.md` + `index.md` |
| 章节导航 | `<章节中文名>.md`（同时创建 `index.md`） | `算法分析.md` + `index.md` |
| 原子笔记 | `<X.Y> <名称>.md` | `1.1 复杂度分析.md` |
| 原始笔记 | 放在学科 `notes/` 目录 | `CSAPP/notes/1.md` |

> ⚠️ 原子笔记**必须**包含 `X.Y` 数字前缀，否则无法在 Explorer 中正确排序。

## 原子笔记内容要求

每篇原子笔记应遵循以下格式：

```markdown
---
created: <YYYY-MM-DD>
tags: [<章节标签>, <学科标签>, <知识标签>]
---

# <知识点名>

用一两句话概述该知识点的核心内容。

## <子标题>

- 要点 1
- 要点 2

## 相关概念

- 可选的跨章节/跨学科链接（使用 markdown 链接，不要用 wiki-link）
```

**要点**：
- **一个笔记只讲一个概念**，粒度要均匀，避免大而全
- 公式使用 LaTeX 语法（`$$` 或 `$`），Quartz 会自动渲染
- 代码块标注语言类型
- 图片放在章节目录下的 `images/` 或学科公共 `images/`

## 章节导航文件（index.md + Hub）

每个章节目录中需要两个内容一致的文件：

- `index.md` — **网站使用的版本**，所有 wiki-link 应指向此文件
- `<章节名>.md` — Hub 文件，保留在 Git 中供本地 Obsidian 使用（不上传至网站）

**格式**（两个文件内容一致）：

```markdown
---
created: <YYYY-MM-DD>
tags: [<学科标签>, moc, <章节标签>]
---

# 第<N>章 <章节中文名>

> 一句话概述本章内容

## 本章知识点

- [[<目录>/<X.Y> <知识点>]]
- [[<目录>/<X.Y> <知识点>]]
- ...

---

[[../index|← 返回 <学科> 总览]]
```

> ⚠️ 返回链接请使用 `../index` 指向上级目录的 `index.md`，而不是指向 Hub 文件。
>
> ⚠️ **所有指向 `index.md` 的 wiki-link 必须带显示名**，否则图谱中会显示无意义的 "index" 节点：
> - ✅ `[[../index|CSAPP总览]]`
> - ❌ `[[../index]]`
>
> 在 Explorer 中，原子笔记显示 `X.Y` 数字前缀（如 `1.1 复杂度分析`），`index.md` 文件显示其 H1 标题（如 `第1章 计算机系统漫游`）。

> ⚠️ 知识点列表**必须按 X.Y 编号升序排列**，推荐：`1.01`→`1.02`→`1.10`。

## 学科根节点文件

**位置**：`<学科根目录>/<学科名>.md`
**同时创建**：`<学科根目录>/index.md`（与根节点内容一致，用于 Explorer 点击目录时直达）

```markdown
---
created: <YYYY-MM-DD>
tags: [<学科标签>, map-of-content, root]
---

# <学科名>

> 一句话描述该学科

## 章节导航

| # | 章节 | 核心主题 |
|---|------|---------|
| 第1章 | [[01-<章节>/index\|<章节>]] | 主题词 |
| 第2章 | ... | ... |
```

## 标签体系

| 标签类型 | 格式 | 示例 |
|---------|------|------|
| 学科标签 | 小写英文 | `csapp`、`algorithms` |
| 章节标签 | `<学科>-ch<编号>` | `csapp-ch1`、`algo-ch3` |
| 知识标签 | 中文 | `复杂度`、`浮点数` |
| 页面类型 | `map-of-content` / `moc` / `root` | 根节点用 `root` |

## 提交 PR 流程

1. Fork 本仓库
2. 按照上述规范创建或修改笔记
3. 提交前运行检查：
   - 是否有 `notes/` 文件混入（这些不上传至网站）
   - 原子笔记文件名是否包含 `X.Y` 前缀
   - Hub 文件中知识点是否按编号排序
   - Wiki-link 路径是否正确
4. 提交 Pull Request，说明改动内容

## 注意事项

- 不要上传 PDF 文件（已全局 `.gitignore`）
- 不要在原子笔记中使用出链 wiki-link（会破坏图谱结构），跨章节引用请用 markdown 链接
- `notes/` 目录的原始笔记仅供 Git 备份，不会出现在 Quartz 网站中
- `index.md` 与 Hub 文件必须内容一致，所有 wiki-link 指向 `index.md` 而非 Hub 文件
- **避免自引用**：不要出现 `index.md` 链接到自身，或 Hub 与 `index` 互相引用的情况
- **避免裸 `index` 链接**：指向 `index.md` 的 wiki-link 必须带显示名（如 `[[../index|CSAPP]]`），否则图谱中会出现无意义的 "index" 文本
- 章节导航中知识点列表**必须按 X.Y 编号升序排列**
