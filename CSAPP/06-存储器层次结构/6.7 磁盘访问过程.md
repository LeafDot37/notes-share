# 磁盘访问过程

## 逻辑块号映射

操作系统将磁盘视为连续的**逻辑块号**（Logical Block Number, LBN）序列，由磁盘控制器映射到物理位置（盘面/磁道/扇区）。

## 完整读取过程

1. **CPU 发起请求**：CPU 向磁盘控制器发送命令（逻辑块号、操作类型）
2. **磁盘控制器执行**：控制器将 LBN 转换为物理地址，控制磁盘臂寻道、等待旋转、读写数据
3. **DMA 传输**：数据通过 **DMA**（Direct Memory Access，直接内存访问）从磁盘控制器传输到主存，**无需 CPU 参与**
4. **中断通知**：DMA 完成后，磁盘控制器通过**中断**通知 CPU
5. **CPU 恢复执行**：CPU 处理中断，继续执行后续指令

## DMA 的优势

- DMA 期间 CPU 可以执行**其他指令**
- 实现 CPU 与 I/O 的**并行工作**
- 显著提升系统吞吐量

## 关键时间点

```
CPU 发起请求 → 磁盘寻道 → 旋转延迟 → 数据传输
     ↓                              ↓
   CPU可做其他工作             DMA 传输+中断通知
```
