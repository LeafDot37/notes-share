# 虚拟内存

# 概念

虚拟寻址与物理寻址，物理寻址指直接在物理存储（磁盘）上寻找地址对应的内容并返回，虚拟地址指CPU先生成一个虚拟地址，接着经过内存管理单元MMU利用主存中的查询表翻译成物理地址，再寻找内容并返回

![1778320081211](images/9/1778320081211.png)

![1778320088456](images/9/1778320088456.png)

使用虚拟寻址的好处是有效性和独立性

![1778320206362](images/9/1778320206362.png)

虚拟内存的本质就是通过MMU和操作系统合作，利用DRAM作为磁盘上庞大虚拟地址空间的高速缓存，从而为每个进程创造出一个既巨大、独立又高效的内存环境。

虚拟内存狭义上可以理解为虚拟地址所组成的内存，并不是实际存在的，而是在物理内存和磁盘中不断交换，而物理内存指的是主存（DRAM）上的空间，而磁盘上对应的内存空间通常叫做交换空间，用来存储过量的进程文件。

虚拟地址是CPU生产的地址，既不是磁盘地址也不是内存地址，而物理地址指的是数据在主存上的实际地址

# 页表

![1778321251767](images/9/1778321251767.png)

PTE放在DRAM中，存储物理内存和虚拟内存的映射关系

虚拟内存并不是单纯存在于磁盘上，而是物理内存和磁盘空间的总和。可以把它理解为一个由虚拟地址构成的那个巨大的逻辑空间，它由以下两部分组成：

* 物理内存（DRAM）：存放当前正在使用的数据。
* 磁盘空间（交换空间）：存放暂时不用的数据。

## 页命中

![1778321377996](images/9/1778321377996.png)

当 CPU 想要读包含在 VP 2 中的虚拟内存的一个字时会发生什么，VP2 被缓存在 DRAM 中。使用地址翻译技术，地址翻译硬件（MMU）将虚拟地址作为一个索引来定位 PTE 2, 并从内存中读取它。因为设置了有效位，那么地址翻译硬件就知道VP2是缓存在内存中的了。所以它使用 PTE 中的物理内存地址（该地址指向 PP1中缓存页的起始位置），构造出这个字的物理地址。

## 缺页（页不命中）

![1778321589572](images/9/1778321589572.png)

![1778321698326](images/9/1778321698326.png)

![1779030286860](images/9/1779030286860.png)

CPU 引用了 VP 3 中的一 个字，VP 3 并未缓存在DRAM 中。地址翻译硬件从内存中读取 PTE 3, 从有效位推断出 VP 3 未被缓存，并且触发一个缺页异常。缺页异常调用内核中的缺页异常处理 程序，该程序会选择一个牺牲页，在此例中就是存放在 PP 3 中的 VP 4。如果 VP 4 已经被修改了，那 么内核就会将 它复制回磁盘（写回模式）。无论哪种情况，内核都会修改 VP 4 的页表条目，反映出 VP 4 不再缓存在主存中这一事实。

接下来，内核从磁盘复制 VP 3 到内存中的 PP3, 更新 PTE 3，随后返回。当异常处理程序返回时，它会重新启动导致缺页的指令，该指令会把导致缺页的虚拟地址重发送到地址翻译硬件。但是现在，VP 3 已经缓存在主存中了，那么页命中也能由地址翻译硬件正常处理了。

# 作用

## 虚拟内存作为缓存工具

![1778321025758](images/9/1778321025758.png)

![1778321225634](images/9/1778321225634.png)

这里所说的概念上而言，虚拟内存被组织为一个由存放在磁盘上的N个连续的字节大小的单元组成的数组，并不是说虚拟内存就是磁盘上的一个数组，这只是逻辑形式的描述，可以理解成磁盘上对虚拟内存的备份，因为理论上虚拟内存的数据都存在于磁盘上。

实际上，对这个数组进行拆分，前半部分（代码段、全局变量、动态库等）指的是磁盘上的可执行文件，后半部分指的是运行时的交换空间（堆、栈中不常用的数据），当物理内存不够用时，才会被临时踢到交换区，而剩下的未分配即是申请的虚拟内存中未被占用的部分

而这些磁盘上的数据被缓存到物理内存中。

这里的“数组”是纯粹的逻辑抽象，在物理实现上，数据依然是分散存储的。这里被当作数组的原因是可以依靠页表和MMU将分散的数据映射成连续的索引。

## 虚拟内存作为内存管理的工具

![1779032131050](images/9/1779032131050.png)

![1779033633016](images/9/1779033633016.png)

![1779033639002](images/9/1779033639002.png)

这里三个核心观点分别理解为

每个虚拟内存页面都要被映射到一个物理页面：虚拟内存页面必须被映射到物理页面，也即缓存到主存中，才能被CPU调取，如果未被映射则触发缺页异常，操作系统将映射设置好才可继续进行

一个虚拟内存页面每次可以被分配到不同的物理页面：这是指对于不同的时间，虚拟内存页面可能分配到不同的物理页面，强调了虚拟内存的灵活性

不同的虚拟内存页面被映射到相同的物理页面：核心是共享，即不同的虚拟内存可以访问同一个物理内存内容，这里要注意物理内存中存放的内容是共享的

## 基于页表的地址翻译

![1779088284633](images/9/1779088284633.png)

这里要区分概念，页面大小是针对于系统来说的，无论虚拟内存还是物理内存都分成了一个个页面，页面大小就是每个页面的大小。因此虚拟页偏移量和物理页偏移量就是用来找到数据在页面的具体位置，是页面大小的log关系。而虚拟页号跟页表的个数有关，由于页表要将所有的虚拟地址和物理地址进行对应，因此虚拟页号也是页表个数的log关系

## TLB加速（翻译后备缓冲器）

![1779088753675](images/9/1779088753675.png)

![1779088809998](images/9/1779088809998.png)

![1779088817399](images/9/1779088817399.png)

![1779088948538](images/9/1779088948538.png)

这里需要注意，TLB存储的是PTE对应的内容，即PPN和有效位等，同时对于页表，其是利用VPN做索引，得到的那块内容叫做PTE。

所以，使用TLB时，输入VPN，如果命中则返回PTE，得到PPN，如果不命中，则需要从内存中的页表中查找VPN，得到对应的PTE，拿出PPN，进而组成PA即物理地址。

碎片化会导致内存利用率降低，分为内部碎片和外部碎片

内部碎片
▪ 对一个给定块, 当有效荷载小于块的大小时会产生
内部碎片
▪ 无法被垃圾回收，只能靠free回收
外部碎片
▪ 是当空闲内存合计起来足够满足一个分配请求，但是没有一个独立的空闲块足够大可以来处理这个请求时发生的。
▪ 可以被合并回收
