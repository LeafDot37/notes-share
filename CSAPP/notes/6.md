# 总览

## 各类存储器

![1777826121448](images/6/1777826121448.png)

RAM，随机访问存储器，又分为静态RAM（SRAM）和动态RAM（DRAM）

SRAM依靠双稳态特性，有电就能保持值，非常稳定，更快更贵

DRAM依赖电容充放电，对干扰敏感，电压被扰乱就无法恢复，漏电，因此需要刷新重写数据保持数据不丢失

对于一个DRAM芯片，连接到内存控制器，通过行缓冲区，将该行直接复制过去，接着再在缓冲区中寻找所需数据

![1777826646030](images/6/1777826646030.png)

对于一个内存模块，由多个DRAM芯片组成将ij坐标广播到每个DRAM芯片上并取出值，进而得到目标数据的值

![1777826768424](images/6/1777826768424.png)

DRAM也出现了增强后的变种，比如DDR SDRAM，双倍数据速率同步DRAM，会比正常的DRAM快得多

![1777827371766](images/6/1777827371766.png)

简单来说，

存储器的读操作是，CPU先将目标地址放到总线，存储器根据总线上的地址找到对应数据并放到总线上，CPU再从总线上获取目标数据并放入寄存器

存储器的写操作是，CPU先将目标地址放到总线，存储器根据总线上的地址找到目标位置并等待数据，CPU再将目标数据放到总线上，存储器从总线上获取目标数据存入目标地址

## 访问过程

![1777910355929](images/6/1777910355929.png)

通过磁盘控制器，将逻辑块号映射为盘面、磁道、扇区的三元组信息，接着通过寻道、旋转、数据传输的过程，将物理扇区内的信息提取，放到内存中

![1777910828691](images/6/1777910828691.png)

当 CPU 发出了请求之后，在磁盘执行读的时候，它通常会做些其他的工作。回想一下，一个 1GHz 的处理器时钟周期为 Ins,在用来读磁盘的 16ms 时间里，它潜在地可能执行 1600 万条指令。在传输进行时，只是简单地等待，什么都不做，是一种极大的浪费。
在磁盘控制器收到来自 CPU 的读命令之后，它将逻辑块号翻译成一个扇区地址，读该扇区的内容，然后将这些内容直接传送到主存，不需要 CPU 的干涉（图 6-12b)设备可以自己执行读或者 写总线 事务而不需要 CPU 干涉的过程，称为 直接内 存访问（DirectMemory Access, DMA)。这种数据传送称为 DMA 传送（DMA transfer）。

在 DMA 传送完成 ，磁盘扇区的内容被安全地存储在主存中以后 ，磁盘控制器通过给CPU 发送一个中断信号来通知 CPU(图 6-12c)基本思想是中断会发信号到 CPU 芯片的一个外部引脚上。这会导致 CPU 暂停它当前正在做的工作，跳转到一个操作系统例程。这个程序会记录下 I/O 已 经完成 ，然后将控制返回到 CPU 被中断的地方。

## 固态硬盘（SSD）

固态硬盘(Solid State Disk,SSD)本质也是磁盘，但是技术上不是磁盘，而是基于闪存技术实现

![1777910623339](images/6/1777910623339.png)

![1777910698401](images/6/1777910698401.png)

除了读写操作还有擦除操作，这是由闪存的性质决定的，闪存写入只能从1变成0而不能从0变成1，所以一个page在写入数据之前各位都为1，写入过程本质是将某些存储位从1变成0。

擦除是指将存储位都变成1，擦除是以块为单位，擦除会减少寿命

顺序访问比随机访问快，写操作比读操作慢

## 缓存命中/不命中

缓存命中指，需要第k+1层的某个数据，这个数据被放到了k层的缓存中，因此可以直接在缓存中拿出数据块，被称作缓存命中

![1777912643332](images/6/1777912643332.png)

缓存不命中指需要第k+1层的某个数据，这个数据没有在k层的缓存中，因此需要k层从k+1层中拿出该数据。之后会根据两个策略（放置策略和替换策略）将该数据放到k层中。

## 高速缓存存储器

![1777913029931](images/6/1777913029931.png)

位于寄存器和内存之间充当缓存

![1777913140874](images/6/1777913140874.png)

![1777913356103](images/6/1777913356103.png)

![1777913381456](images/6/1777913381456.png)

简单来说，通过将地址按照t、s、b的映射，将一个地址映射到缓存中，t对应找line（使用tag验证是否是正确的数据），s对应找set，b对应找block内的偏移，接着通过valid验证，检测这个位置的数据是否有效，进而判断缓存是否命中

> 注意，b偏移指的是字节偏移，因为b是来自于一个块内有2^b个字节，一个block中存放不只一个数据，偏移可以做到这一点

在这里不适用高位进行set映射是因为如果使用高位进行set映射，会将连续的数据映射到同一个set中，这样就会多次更改set内的line，由于line的数量一般较少，所以容易发生对一个set内的line多次更改的情况，使得cache利用率低

> 这里存在着一个问题，会不会有这么一种情况，假如块大小为64字节，b为6，也就是会截取目标地址的后6位，当如果后6位是111111，那么偏移为63，这时候目标只能存在块的最后一个字节，但是如果目标数据的数据类型是大于1个字节表示的（比如int），那么不就会出问题吗。
>
> 答案是不会，因为内存上有这么一个规范，数据的地址位数据大小的倍数，因此后6位内一定包括了这个信息，这会使得至少流出数据大小的空间

> 这里还有另一个问题，在读取时怎么确定停止的位置，实际上，在读取时并不确定停止的位置，而是将整个cache line都传输，交给CPU根据类型判断结束位置

块是内存传输的最小单位，在将上一级存储上的数据传给这一级时，会将块内容填满。比如块有64字节，这一级需要一个double数组的某个值，且并不在这级存储中，那么会在上一级存储中找到这个值，并将其和其周围的共计8个数据放入块中都进行传输

## 重新排列提高空间局部性

由于数据放入缓存时，会将空间周围的数据一起放入cache line，于是，尽可能将下一步需要的数据一起放入缓存，就可以显著提升性能

![1777963705313](images/6/1777963705313.png)

![1777964109769](images/6/1777964109769.png)

### 第一梯队：缓存友好型 (kij, ikj)

这是性能最好的顺序。以 `ikj` 为例：

```c
for (i = 0; i < N; i++)
  for (k = 0; k < N; k++) {
    r = A[i][k]; // 1. 读取 A 的一个元素
    for (j = 0; j < N; j++)
      C[i][j] += r * B[k][j]; // 2. 读取 B 的一行，更新 C 的一行
  }
```

* **分析**：
  * **A (`A[i][k]`)**：在内层循环 `j` 变化时，`A` 的索引不变。它被提取到寄存器 `r` 中，**复用性极高**。不命中率 $\approx 0$。
  * **B (`B[k][j]`)**：`j` 递增，意味着**按行扫描**。这是顺序访问！每读 4 个元素才发生 1 次不命中。不命中率 = **0.25**。
  * **C (`C[i][j]`)**：`j` 递增，也是**按行扫描**。同样是顺序访问。不命中率 = **0.25**（写操作通常也遵循类似的分配策略）。
* **总结**：每次内层迭代的不命中率总和 $ = 0 + 0.25 + 0.25 = \mathbf{0.5}$。这是理论上的最优解之一。

### 第二梯队：勉强及格型 (ijk, jik)

这是教科书中最常见的写法，但性能一般。以 `ijk` 为例：

```c
for (i = 0; i < N; i++)
  for (j = 0; j < N; j++) {
    sum = 0.0;
    for (k = 0; k < N; k++)
      sum += A[i][k] * B[k][j]; // 1. A 按行读，B 按列读
    C[i][j] = sum;
  }
```

* **分析**：
  * **A (`A[i][k]`)**：`k` 递增，**按行扫描**。顺序访问。不命中率 = **0.25**。
  * **B (`B[k][j]`)**：`k` 递增，意味着访问 `B[0][j], B[1][j], B[2][j]...`。这是**按列扫描**！
  * **C (`C[i][j]`)**：在最内层循环中，`C[i][j]` 的位置不变，它只是被反复读写（通常在寄存器或L1中），不命中率 $\approx 0$。
* **总结**：每次内层迭代的不命中率总和 $= 0.25 + 1.0 + 0 = \mathbf{1.25}\ $。比最优解慢了 2.5 倍。

### 第三梯队：灾难型 (jki, kji)

这是性能最差的写法。以 `jki` 为例：

```c
for (j = 0; j < N; j++)
  for (k = 0; k < N; k++) {
    r = B[k][j]; // 1. B 按列读
    for (i = 0; i < N; i++)
      C[i][j] += A[i][k] * r; // 2. A 按列读，C 按列读
  }
```

* **分析**：
  * **B (`B[k][j]`)**：在最内层循环中保持不变，复用性好，不命中率 $\approx 0$。
  * **A (`A[i][k]`)**：`i` 递增，**按列扫描**。步长巨大，**每次未命中**。不命中率 = **1.0**。
  * **C (`C[i][j]`)**：`i` 递增，**按列扫描**。步长巨大，**每次未命中**。不命中率 = **1.0**。
* **总结**：每次内层迭代的不命中率总和 = $1.0 + 0 + 1.0 = \mathbf{2.0}$。这是最慢的，比最优解慢了 **4 倍**。

3B^2<C的要求保证了缓存内能同时放下ABC的块

由于数据进缓存最小单位是缓存行，一行有8double，所以由于缓存一开始内部是没有数据的，一次读取8个double，这8个都用之后出现一次不命中，于是有B^2/8的每块不命中率

总不命中次数等于单次不命中次数×迭代次数

总共迭代（n/B）^3次，单次不命中次数为B^2/8 *2

于是总不命中次数为![1777999069602](images/6/1777999069602.png)

分块的本质是将矩阵分成能完全放入缓存的块，这样可以保证在计算块内时，缓存内的数据永远可复用，进而提高时间局部性。具体来说，在块内计算时，取A的一个值时，会将这行拿入缓存，取B的一个值时，也会将这行拿入缓存，A按行取，B按列取，这样可以在拿B列的同时顺便把行也都拿入缓存。
