# 内存性能

## 加载延迟（Load Latency）

从内存加载数据到寄存器的延迟通常为 **4 个周期**（L1 缓存命中时）。加载指令的结果需要经过多个流水线阶段才能使用。

## 存储转发（Store-Forward）

当加载指令读取一个刚刚写入的内存地址时，处理器可以通过**存储转发**机制直接将存储缓冲区的数据转发给加载指令，无需等待写回缓存。

```
存储指令: store r1 -> [addr]
加载指令: load [addr] -> r2    ← 从存储缓冲区直接转发
```

## 读写相关（Read-After-Write Dependency）

- 读操作依赖于之前写操作的完成
- 如果地址相同，加载必须等待存储完成
- 但不同地址的存储和加载可以并行执行

## 对优化的影响

- 使用减少内存访问
- 将数据加载到寄存器中操作
- 合理安排读写顺序减少相关停顿
